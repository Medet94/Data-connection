Часть 1: Введение в подключение данных

Что такое подключение данных?

Подключение данных — это приложение в Phoenix, которое помогает перенести ваши данные из внешних систем в Phoenix. Представьте его как мост между вашими существующими данными (хранящимися в базах данных, облачных хранилищах или других системах) и Phoenix, где вы можете анализировать и работать с этими данными.

Простыми словами: Подключение данных позволяет подключиться к вашим данным, где бы они ни находились, и перенести их в Phoenix для использования.

Зачем использовать подключение данных?

  • Доступ ко всем вашим данным в одном месте: Вместо переключения между различными системами для доступа к разным данным, перенесите все в Phoenix
  • Автоматическое обновление данных: Настройте регулярные обновления, чтобы ваши данные в Phoenix оставались актуальными
  • Сохранение безопасности и разрешений: Подключайтесь к внешним данным, сохраняя при этом надлежащие средства контроля безопасности
  • Без программирования: Используйте дружественный интерфейс вместо написания сложного кода для подключения источников данных

Что вы можете делать с подключением данных

  • Подключаться к базам данных, таким как Snowflake, PostgreSQL или SQL Server
  • Извлекать данные из облачных хранилищ, таких как Amazon S3 или Google Cloud Storage
  • Импортировать файлы из локальных систем
  • Подключаться к веб-данным через API
  • Настраивать автоматические регулярные обновления данных
  • Предварительно просматривать данные перед их переносом в Phoenix

Часть 3: Понимание ключевых концепций

Три ключевых термина, которые вам нужно знать

1. Источники

Источник — это просто то место, где в настоящее время находятся ваши данные. Это может быть:

  • База данных, такая как PostgreSQL или Snowflake
  • Облачное хранилище, такое как Amazon S3
  • Файлы на сервере
  • Веб-сервис с API

Думайте об источнике как о "что" — какая система содержит ваши данные?

2. Подключения

Подключение — это то, как Phoenix достигает вашего источника данных. Есть два основных типа:

Прямое подключение:

[Изображение: Прямое подключение между Phoenix и источником данных]

Figma node ID: 13058:69615

Когда использовать прямые подключения:

  • Лучше всего для источников данных в публичных сетях
  • Автоматически управляется Phoenix
  • Требует источник из разрешенного списка
  • Не может подключаться к частным сетям или локальным источникам данных
Подключение через агент:

[Изображение: Подключение через агент между Phoenix и источником данных]

Figma node ID: 13058:69642

Когда использовать подключения через агент:

  • Может подключаться как к публичным сетям, так и к частным/локальным источникам данных
  • Требует ручных обновлений или обслуживания
  • Требует агента, установленного на хосте
  • Хороший вариант, когда ваши данные находятся за брандмауэром или в частной сети
3. Синхронизации

Синхронизация определяет, какие конкретные данные вы хотите перенести в Phoenix и как часто. Например:

  • Конкретная таблица из базы данных
  • Папка файлов из облачного хранилища
  • Данные, соответствующие определенным критериям

Думайте о синхронизации как о "какие" — какие именно данные вы хотите и когда?

Часть 6: Создание и управление синхронизациями

Что такое синхронизация?

После подключения к источнику данных вам нужно настроить синхронизацию, чтобы указать именно те данные, которые вы хотите перенести в Phoenix, и как часто.

Думайте об этом следующим образом:

  • Ваше подключение к источнику подобно наличию ключа от библиотеки
  • Синхронизация — это выбор конкретных книг для переноса в Phoenix и решение, как часто проверять новые издания

Типы синхронизаций

Phoenix Data Connection предлагает несколько типов синхронизаций для соответствия различным потребностям в данных:

1. Пакетная синхронизация

Самый распространенный тип - извлекает данные по расписанию:

  • Пакетная синхронизация таблиц: Извлекает данные из таблиц баз данных (PostgreSQL, Snowflake и т.д.)
  • Пакетная синхронизация файлов: Извлекает файлы из систем хранения (S3, Google Cloud Storage и т.д.)
2. Потоковая синхронизация

Для непрерывных потоков данных в реальном времени (для систем, которые поддерживают потоковую передачу)

3. Захват изменений данных (CDC)

Извлекает только данные, которые изменились с последней синхронизации (более эффективно для больших наборов данных)

4. Медиа синхронизация

Для специализированного медиа-контента, такого как изображения, видео или аудиофайлы

Часть 8: Глоссарий терминов

А-В

  • Агент: Небольшая программа, установленная в вашей сети, которая помогает Phoenix подключаться к вашим данным, особенно полезна для доступа к данным за брандмауэрами или в частных сетях
  • Подключение через агент: Метод подключения, который использует агента для доступа к источникам данных в частных сетях или за брандмауэрами
  • Источник API: Подключение к интерфейсу программирования приложений внешней системы, который позволяет получение данных
  • Добавление: Тип транзакции, где каждая синхронизация добавляет новые данные без удаления существующих данных
  • Приложение: Логический контейнер, который организует связанные источники данных и подключения
  • Пакетная синхронизация: Процесс, который перемещает данные от источника в Phoenix по расписанию
  • Навигация хлебных крошек: Элемент пользовательского интерфейса, показывающий ваше текущее местоположение в иерархической структуре

Г-Д

  • CDC (Захват изменений данных): Метод синхронизации, который передает только данные, которые изменились с последней синхронизации
  • Сопоставление столбцов: Процесс определения того, как столбцы исходных данных будут переведены в столбцы Phoenix
  • Настройки подключения: Детали конфигурации для того, как Phoenix подключается к внешним источникам данных
  • Тип подключения: Протокол или метод, используемый для подключения к источнику данных (JDBC, REST, S3 и т.д.)
  • Создать пакетную синхронизацию: Кнопка, которая инициирует процесс настройки новой синхронизации данных
  • Предварительный просмотр данных: Функция, которая показывает образцы строк и столбцов из вашего источника перед созданием синхронизации
  • Прямое подключение: Когда Phoenix подключается напрямую к источнику данных без агента

Е-И

  • Вкладка редактирования синхронизаций: Интерфейс для просмотра существующих синхронизаций и создания новых в деталях источника
  • Политика исходящего трафика: Правила, которые контролируют, как данные могут вытекать из Phoenix
  • Интерфейс исследования: Пользовательский интерфейс, который позволяет просматривать, предварительно просматривать и выбирать данные для синхронизации
  • Обертка поля: Символы (такие как кавычки), используемые для инкапсуляции полей в структурированных текстовых файлах
  • Разделитель полей: Символ, используемый для разделения значений в структурированных текстовых файлах (запятая, табуляция и т.д.)
  • Синхронизация файлов: Синхронизация, которая передает файлы, а не структурированные данные
  • Инкрементальная синхронизация: Синхронизация, которая приносит только новые или измененные данные

К-П

  • Состояние загрузки: Визуальная обратная связь, показанная, когда Phoenix подключается к вашему источнику данных
  • Вкладка сопоставления: Интерфейс для настройки того, как исходные столбцы сопоставляются с целевыми столбцами
  • Синхронизация медиа: Специализированный тип синхронизации для работы с медиа-контентом, таким как изображения, видео или аудиофайлы
  • Управление мульти-синхронизацией: Интерфейс для выбора и управления несколькими синхронизациями вместе
  • Вкладка обзора: Сводный вид, показывающий детали подключения источника и метрики
  • Местоположение вывода: Путь папки, где синхронизированные данные сохраняются в Phoenix
  • Предварительный просмотр: Функция, которая показывает образцы данных перед созданием полной синхронизации
  • Первичный ключ: Столбец или набор столбцов, которые уникально идентифицируют каждую строку в таблице
  • Проект: Организационный контейнер в Phoenix для связанных ресурсов

Р-Ф

  • Запрос: Запрос на конкретные данные, часто написанный на SQL
  • Ресурс: Индивидуальная сущность данных (таблица, файл, конечная точка API), доступная в источнике
  • Браузер ресурсов: Левая боковая панель, которая показывает доступные ресурсы в вашем источнике данных
  • Схема: Структура ваших данных (столбцы, типы данных и т.д.)
  • Умное обнаружение: Автоматический анализ данных для определения подходящих типов
  • Синхронизация снимка: Синхронизация, которая заменяет все предыдущие данные новыми данными
  • Источник: Внешняя система, где ваши данные изначально находятся
  • Мастер создания источника: Пошаговый интерфейс для настройки новых подключений к данным
  • Страница деталей источника: Основной интерфейс для управления существующим источником данных
  • Тип источника: Категория источника данных (База данных, API, Облачное хранилище и т.д.)
  • Потоковая синхронизация: Непрерывный поток данных от источника к Phoenix в режиме реального времени
  • Интерфейс создания синхронизации: Пользовательский интерфейс, где вы настраиваете, как данные перемещаются от источника к Phoenix
  • Тип синхронизации: Метод, используемый для передачи данных (Пакетный, Потоковый, CDC и т.д.)
  • Панель деталей таблицы: Интерфейс для настройки сопоставлений столбцов и других настроек таблицы
  • Тестирование подключения: Функция для проверки действительности настроек подключения перед сохранением
  • Тип транзакции: Как обрабатываются обновления данных (Снимок, Добавление, Обновление)

Заключение

Это руководство предоставило исчерпывающий обзор приложения подключения данных в Phoenix, разработанный для нетехнических пользователей. Следуя этим инструкциям, вы должны быть способны подключаться к базам данных PostgreSQL и хранилищу S3, создавать и управлять синхронизациями, а также отслеживать ваши потоки данных.

По мере продолжения работы с подключением данных обращайтесь к этому руководству всякий раз, когда вам нужно освежить в памяти конкретные шаги или концепции.

Основные преимущества подключения данных:

  • Централизованный доступ ко всем вашим данным в Phoenix
  • Автоматизированные и планируемые обновления данных
  • Безопасные подключения с сохранением разрешений
  • Интуитивный интерфейс без необходимости программирования
  • Гибкие опции синхронизации для различных потребностей данных

Если у вас возникнут вопросы или проблемы при работе с подключением данных, обратитесь к соответствующим разделам этого руководства или свяжитесь со службой поддержки Phoenix.
